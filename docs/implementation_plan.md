# DeepFake Detection System Implementation Plan

## 1. Goal Description
The objective is to build a **Trust Infrastructure** rather than just a classification model. The goal is machine-verifiable authenticity with probabilistic confidence.
**Core Constraint:** Human perception fails at scale.
**Failure Modes:** Adversarial generators, re-encoding/compression, platform metadata stripping.
**Ultimate Goal:** Shift from "Is this fake?" to "Can this be proven real?".

> [!IMPORTANT]
> **Measurement of Success:** The goal is not certainty, but "measurable, auditable doubt".

## 2. Layer 0: Threat Modeling (Mandatory First Step)
Before writing code, you must define the attacker. skipping this causes system collapse.
3.  **Attacker Capabilities:**
    *   **Generators:** Access to SOTA (Stable Diffusion, Midjourney, Sora, Runway).
    *   **Post-Processing:** Ability to resize, crop, recompress, and strip metadata.
    *   **Adversarial attacks:** Ability to train specifically against your detectors.
    *   **Metadata:** Ability to remove or spoof standard metadata.

## 3. Layer 1: Data Acquisition (Ground Truth)
**Do not train on a single dataset.** You need four distinct categories.
*   **Dataset A: Human-Authored:** Verified real media from trusted sources.
*   **Dataset B: Known-Model Synthetic:** Generated by SD, Midjourney, etc.
*   **Dataset C: Unknown-Model Synthetic:** Open-source models + fine-tuned variants.
*   **Dataset D: Adversarial:** Media from A/B/C that has been explicitly attacked (noise, blur, recompressed).

**Data Ingestion Rules:**
*   Retain **Original Bytes**.
*   Retain **Post-processed variants**.
*   **Transformation History:** Log every step done to the data.
*   *No labels = No learning.*

## 4. Layer 2: Signal Extraction (The Forensics)
**Do not rely on visual realism.** Use forensic artifacts that are sub-perceptual.

### Image Signals
*   **Frequency Domain:** FFT (Fast Fourier Transform), DCT (Discrete Cosine Transform) analysis for spectral anomalies.
*   **PRNU:** Photo Response Non-Uniformity (Sensor noise patterns) - valid for camera-originated media.
*   **Texture Entropy:** Patch-level anomalies in high-frequency texture.
*   **Diffusion Residues:** Specific noise patterns left by diffusion upsampling steps.
*   **Edge Gradients:** Over-smoothed edges characteristic of generative upscalers.

### Video Signals
*   **Temporal Coherence:** Are artifacts consistent across frames?
*   **Optical Flow:** Jitter analysis in motion vectors.
*   **Noise Inconsistency:** Does frame-to-frame noise distribution match physics?
*   **Mouth-Phoneme Desync:** Even visually aligned lips may fail micro-timing against the audio track.

### Audio Signals
*   **Phase Coherence:** Errors in waveform phase continuity.
*   **Spectral Regularization:** Look for "over-perfect" spectral structures.
*   **Breath/Pause Detection:** Absence of natural breathing sounds.
*   **Micro-Prosody:** Flattening of subtle pitch variations.

## 5. Layer 3: Modeling (Ensemble Architecture)
**Never rely on a single model.** Build an ensemble.

### The Ensemble Members
1.  **Spatial:** CNN (EfficientNet) or ViT (Vision Transformer) for spatial artifact detection.
2.  **Temporal:** Temporal Transformer / 3D-CNN for video consistency.
3.  **Audio:** Audio Transformer (e.g., Wav2Vec features) for speech forensics.
4.  **Statistical Baseline:** Non-NN approach (e.g., SVM on frequency features) as a sanity check.
5.  **OOD Detector:** Explicitly detect "Out of Distribution" inputs (unknown generators).

### Outputs
*   **Probability Score**
*   **Confidence Interval** (How sure is the model?)
*   **Feature Attribution** (Which pixels/frames triggered the verdict?)

## 6. Layer 4: Adversarial Robustness Loop (Continuous)
The model lifespan is ~3 months without this.
1.  **Red Teaming:** Continuously train generators *against* your frozen detector.
2.  **Feedback:** Add the successful "bypasses" back into the training set (Dataset D).
3.  **Rotation:** Periodically rotate model architectures to invalidate attacker priors.
4.  **Inference Defense:** Introduce random feature dropout during inference to prevent gradient estimation by attackers.

## 7. Layer 5: Provenance (The "Trust" Layer)
Detection comes late; Provenance comes first.
1.  **Cryptographic Signing:** Sign media at creation time (if possible).
2.  **Hash Chaining:** Link edits to the original via hash chains.
3.  **Binding:** Bind media to device or model identity.
4.  **Tamper-Evident Logs:** Use a ledger (immutable log) for all checks.

> [!NOTE]
> Shifts the question from "Is this fake?" to "Can we prove this is real?". Absence of proof is a strong signal.

## 8. Layer 6: System Output Interface
**Never output a binary label.** Binary labels invite lawsuits and misuse.
*   **Authenticity Score:** 0-100 scale.
*   **Source Confidence:** High/Medium/Low based on provenance.
*   **Modification Likelihood:** "Likely modified by X".
*   **Match Probability:** "95% likelihood generated by Midjourney v5".

## 9. Layer 7: Platform Integration
Standalone tools fail. Integration is key.
*   **Ingestion Points:** Social media uploads, Messaging apps, CMS.
*   **Latency Budget:**
    *   Images: < 500 ms
    *   Short Video: < 2 seconds
*   **Scalability:** Must handle platform-scale throughput.

## 10. Layer 8: Evaluation Metrics (Non-Negotiable)
Accuracy is a bad metric. Use:
*   **False Negative Rate under Attack:** How often do we miss a targeted attack?
*   **Robustness:** Performance after recompression/resizing.
*   **Generalization:** Performance on *unseen* generators.
*   **Calibration Error:** Does "90% confident" actually mean 90% correct?
*   **Decay:** Track performance degradation over time.

## 11. Layer 9: Legal & Societal Interface
*   **Audit Logs:** Every decision must be logged.
*   **Explainability:** "Why" confidence is low/high.
*   **Appeals:** Mechanism for users to challenge a flag.
*   **Language:** Use terms like "Evidence of manipulation" rather than "Fake".

---

## Technical Stack Recommendations
*   **Preprocessing:** OpenCV, FFmpeg (forensic extraction)
*   **ML Framework:** PyTorch (Ensembles, custom loss functions)
*   **Provenance:** C2PA / Content Credentials SDK
*   **Storage:** SQL (metadata), Object Storage (media versions)
*   **Serving:** ONNX / TensorRT for latency targets

## Summary
This is a hybrid solution: **Forensics + ML + Crypto + Platform Engineering**.
Perfect detection is impossible. The goal is to establish a **Chain of Trust**.
